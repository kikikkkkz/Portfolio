Mobile App Design

An Android app TagAStreet from design to coding

Overview

TagAStreet is an Android app I developed. Targeting at street style photography, TagAStreet allows users to add tags to their uploaded photos and view photos they want to see with selective tags. With few simple clicks, an outfit picture can be taken, tagged and shared within one single app. By clicking one to as many tags, pictures will be filtered automatically and viewed clearly. The application provides a fast and easy way of keeping as well as seeking daily outfit inspiration for the users.

Process

Proposal

Based on my own experience, I find it hard to filter a collection of images I want, whether using the search function in Google or using the tag in Instagram. Therefore, I came up with an app that can select multiple tags to filter the desired images.

Prototype

Figure 1 - Interactive prototype of UI for filtering pictures

To begin, I drew a sitemap to identify the structure of the app. Then, I sketched out the layout in Sketch and demonstrated tranisition in Flinto. After that, I built the basic framework in Android Studio and started to implement all the functions I proposed. In the end, I tested it out with my friend the received feedbacks to improve.

If a girl brought an oversize denim jacket and she’s wondering how to wear it, she can use TagAStreet. By selecting the tag “denim”, she filtered out the pictures that only relate to denim. After that, if there is “jacket” tag, she can select it as well and the app will filter for her. Lastly, if there exists an “oversize” tag in the refreshed tag list, she can select and get the most precise image she is looking for. In this way, she can easily find the picture she wants. Moreover, if she feels good about her outfit, she can post her outfit picture to the app and contribute to the community as well.

Problems and Solutions

In the beginning, the purpose of the app in my original proposal was to upload street style pictures and share with people. However, at this stage, I only learned how to store images in local storage and did not figure out the web server part. After speaking with the professor, I realized the app can also serve another purpose. Therefore, I changed my main function of the app to “keeping your daily outfit inspo”, which can store the images and view the images in the user’s own phone.

As each tag has different length, a longer tag may ruin the current design. The possible future solution will be using recyclerview to display the tags rather than gridview. So that the appearance of the tags will be flexible in size.

Reflection

For this project, I simply assumed the users' need based on my own experience instead of doing field research. As for designing a real product, I would need to go into the market and do questionnaires or observations to test if my assumption of "users wish to narrow down the image pool by selecting multiple tags" is a problem.
